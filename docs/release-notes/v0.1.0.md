## LDiff v0.1.0

First prebuilt release of **LDiff** — a desktop tool to inspect, compare, and
stage merges between JAR/ZIP archives and folders. Decompiled Java is always
read-only; a merge copies original entry bytes, never the decompiled view.

### Features

- **Inspect** — open a `.jar`, `.zip`, or folder and browse its entry tree with
  lazy index/read, detected text languages, and binary size/CRC/SHA-256 detail.
- **Compare** — CRC tree diff between two archives or folders, Monaco
  source/bytecode diff, text/hex preview, and deep source search.
- **Merge** — stage original entry bytes from one side to the other, review
  pending changes, then save atomically with an optional `.bak` backup.
- **Decompile** — Vineflower source by default, CFR as an alternate engine, and
  ASM bytecode through an isolated JVM sidecar with a bundled Java 17 runtime.

### Install

Download the installer helper for your platform alongside the binary, then:

**macOS** (app is unsigned; the script clears the Gatekeeper quarantine flag):

```bash
bash install-macos.sh
```

**Linux** (Ubuntu 22.04+ / glibc 2.35+):

```bash
bash install-linux.sh LDiff_0.1.0_amd64.AppImage   # portable: ~/.local/bin/ldiff + app menu
bash install-linux.sh LDiff_0.1.0_amd64.deb        # system install via apt
```

See the [README](https://github.com/lyokha113/ldiff#download-and-install-prebuilt)
for details.

### Notes

- macOS build is **arm64 only** and unsigned (ad-hoc). Intel Macs and signed
  builds: build from source.
- Linux build targets **x86_64**, glibc 2.35+ (Ubuntu 22.04). For ARM Linux,
  build from source.
- Linux bundles are unsigned.
